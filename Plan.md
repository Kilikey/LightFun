# 📖 轻小说文库 · 次元书屋 — 项目总规划

> **项目代号**：DimensionBookhouse（次元书屋）  
> **目标定位**：对标 wenku8、真白萌、SF轻小说等主流平台的轻小说在线阅读网站  
> **技术栈**：HTML + CSS + JavaScript（前端渐进式架构，后端可平滑接入）  
> **开发周期**：1个月+  
> **迭代节奏**：每两周发布一个可用版本（MVP → 基础版 → 完整版）  
> **当前路径**：`D:\PythonFile\HTML\test for first html\`  
> **最后更新**：2026-06-15

## 一、项目总目标

将现有单文件 Demo 升级为一个**完整、可长期维护的轻小说阅读平台**，满足：

1. **读者端**：浏览、搜索、阅读、收藏、追更、评论、社交
2. **作者端**：上传作品、管理章节
3. **管理端**：内容审核、运营推荐、数据看板
4. **技术目标**：模块化架构 → 可迁移至 Vue/React → 接入后端 API

## 四、设计原则

| 原则 | 当前做法 | 目标做法 | 为什么 |
|------|---------|---------|--------|
| **数据分离** | JS 数组硬编码在 `library.html` | JSON 文件 + `fetch()` 读取 | 数据可独立维护、后端可无缝替换 |
| **样式分离** | 内联 `<style>` 标签（500+行） | 外链 CSS，按组件/页面拆分 | 样式可复用、可缓存、可主题化 |
| **逻辑分离** | 内联 `<script>` 标签（300+行） | 外链 JS，按职责拆分 | 逻辑可复用、可测试、可多人协作 |
| **组件复用** | 每个页面复制导航栏代码 | 统一 `nav.js` + `nav.html` 模板 | 一改全站生效，减少重复 |
| **状态管理** | 分散的 `localStorage` 直接读写 | 统一的 `state.js` 全局状态 | 状态可追溯、可调试、可同步 |
| **渐进增强** | 纯静态 HTML | 先纯前端 → 后接框架 → 后接后端 | 每步都有可用版本，风险可控 |

## 七、关键决策记录

1. **不使用框架（现阶段）**：用纯 HTML/CSS/JS 先跑通业务逻辑，降低认知门槛，方便随时调试。未来引入框架时，现有模块可直接迁移。
2. **LocalStorage 优先**：用户数据（书架、历史、设置）先存在本地，不依赖后端。后续通过 `user-service.js` 层封装，后端接入时只需替换这一层。
3. **JSON 数据文件**：作品数据从 JS 数组改为 JSON，让前端与数据解耦。未来后端只需替换 JSON 文件的读取路径即可。
4. **主题化 CSS 变量**：所有颜色、字体、间距用 CSS 变量（`var(--color-primary)`）管理，方便切换主题。
5. **1页1文件**：每个页面独立 HTML，用 `<script type="module">` 加载模块化 JS，不引入打包工具。
